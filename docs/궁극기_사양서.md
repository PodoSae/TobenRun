# 궁극기(Ultimate) 코드 사양서

> 대상: MapleStory Worlds 러너 게임 "TobenRun"의 궁극기 시스템
> 작성 기준: 현재 `dev/sjy` 브랜치 소스
> 한 줄 요약: **게이지가 차면 R로 발동 → (컷씬 → 10발 검기 피니셔) → 10초간 전방을 파괴하며 돌진하는 스킬.** 게이지·발동판정·파괴는 서버 권위, 연출·입력은 클라이언트, 그 사이를 `@Sync`가 잇는다.

---

## 1. 구성 파일

| 파일 | 종류 | 역할 |
|---|---|---|
| `RootDesk/MyDesk/Player/UltimateSkill.mlua` | `@Component` (플레이어) | **핵심.** 게이지, 발동 게이트, 돌진 이동효과, 전방 타겟 파괴, 피니셔 트리거 |
| `RootDesk/MyDesk/UI/UltCutsceneHud.mlua` | `@Logic` ClientOnly | 발동 **직전** 컷씬(암전→홀드→플래시) + 피니셔 지연 + 실제 발동 호출 |
| `RootDesk/MyDesk/Skill/UltCutscene.mlua` | `@Logic` ClientOnly | 발동 **순간** 전체화면 스킬 클립(스피릿 칼리버) 1회 재생 |
| `RootDesk/MyDesk/01_Component/PowerStrike.mlua` | `@Component` (플레이어) | 검기 발사·`GetEffectiveDamage` 공유, **`FireFinisherBeams`**(피니셔 10발) |
| `RootDesk/MyDesk/RunGame/RunGameHud.mlua` | `@Logic` ClientOnly | 거리 진행 게이지 충전 요청, 스테이지 시작 시 게이지 리셋 |
| `RootDesk/MyDesk/Monster/MonsterUnit.mlua` | `@Component` (몬스터) | 처치 시 게이지 지급(`GrantUltGauge`), 검기 피격 처리 |
| `RootDesk/MyDesk/Monster/MushmomBoss.mlua` | `@Component` (보스) | 보스전 동안 자동충전 일시정지 + 피격 시 게이지 충전, 돌진 틱뎀 수신 |

## 2. 상태(State) — `UltimateSkill`

### 2.1 동기화 프로퍼티(`@Sync`, 서버→전 클라)

| 프로퍼티 | 타입 | 의미 |
|---|---|---|
| `Gauge` | number | 게이지 수치(0 ~ `MaxGauge`). HUD 게이지 바 구동(PlayerHudUI) |
| `Active` | boolean | **"궁극기 발동 중" 단일 진실 소스.** Slide·자동공격이 읽어 자기 억제 |
| `AutoFillPaused` | boolean | 시간/거리 자동충전 일시정지(보스전에서 true) |

### 2.2 서버 내부 상태

| 프로퍼티 | 의미 |
|---|---|
| `remaining` | 발동 남은 시간(초). 0 이하 → 종료 |
| `boosting` | 이동부스트 스냅샷 1회 가드 |
| `gravityReleased` | 발동 중 점프해서 중력을 되돌렸는지 |
| `bossDmgTimer` | 보스 틱뎀 쿨 카운트다운 |
| `baseWalkSpeed` / `baseFallSpeedMaxX` | 발동 순간 스냅샷한 지상/공중 수평 속도 상한(종료 시 복원) |
| `baseGravity` / `baseFallSpeedMaxY` | `OnBeginPlay`에서 스냅샷한 중력/낙하Y 상한(종료 시 복원) |

> **스냅샷 원칙:** `WalkSpeed`/`FallSpeedMaxX`는 코스 속도가 `RunGameHud`에 의해 begin play 이후 설정되므로 **발동 첫 프레임에** 스냅샷(`boosting` 가드). `Gravity`/`FallSpeedMaxY`는 다른 시스템이 안 건드린 `OnBeginPlay` 시점에 스냅샷(FastFall의 ×4 오염 방지).

## 3. 게이지 충전 (서버 권위, 이벤트 기반)

기본은 **시간 자동충전 OFF**(`GaugeFillPerSec = 0`). 아래 이벤트로만 충전한다(기획 §3.2).

| 소스 | 충전량 | 경로 |
|---|---|---|
| 일반 몬스터 처치 | `KillGaugeGain` = **3%** | `MonsterUnit.GrantUltGauge` → `AddGauge` |
| 거리 진행(≈100m) | `DistanceGaugeGain` = **10%** | `RunGameHud`가 `ultGaugeDistUnits`(≈11u)마다 `RequestDistanceGauge` |
| 보스 피격 | (보스 로직 값) | `MushmomBoss` → `AddGauge` |

- `AddGauge(amount)` (ServerOnly): 발동 중이면 무시, `MaxGauge` 상한. `AutoFillPaused`여도 동작(보스 피격 충전 허용).
- `RequestDistanceGauge` (Server RPC): `AutoFillPaused`면 무시(보스전은 거리 충전 안 함).
- **보스전**: `SetAutoFillPaused(true)`로 거리 충전 차단 → 보스 피격으로만 충전. 보스 사망/제거 시 해제.
- `RequestFillGaugeFull` (Server): 튜토리얼 전용 즉시 완충.

## 4. 발동 흐름 (입력 → 컷씬 → 피니셔 → 돌진)

```
[클라] R키(OnKeyDown, ClientOnly)
   └ RequestActivateFromInput  (클라 선검사: LocalPlayer & not Active & Gauge==Max)
        ├─ _UltCutsceneHud 있음 → UltCutsceneHud:Play(player)
        │       ├ 러너 정지 + 입력 잠금  (SetRunnerActive(false)/LockPlayerInput(true))
        │       ├ OnUpdate 타임라인: 암전(DarkInTime) → 홀드(HoldTime) → 플래시(FlashTime)
        │       └ Finish()
        │           ├ 오버레이 숨김 + 러너/입력 "즉시" 재개  (멈춤 없음)
        │           └ FinisherDelay(0.35s) 뒤 FireFinisher()
        │                 ├ RequestCutsceneFinisher()   → 검기 10발 발사(피니셔)
        │                 └ RequestActivate()            → 실제 발동
        └─ _UltCutsceneHud 없음 → RequestActivate() 직접
                                     │
[서버] RequestActivate (Server, 게이트)
   └ (not Active & Gauge==Max) → Gauge=0, Active=true, remaining=DashDuration(10s)
                                     │ (@Sync Active=true 전파)
[클라] OnSyncProperty("Active", true) → StartDashClient()
   ├ 오라/전방 바람베기 이펙트 부착(전 클라)
   └ (소유 클라) 이동부스트 ApplyBoost + 전체화면 컷씬 _UltCutscene:Play()
```

> **핵심 타이밍 결정(사용자 반복 튜닝 결과):** 컷씬이 끝나면 러너는 **멈추지 않고 즉시 재개**되고, 피니셔 데미지+돌진만 `FinisherDelay`(0.35초) 뒤에 실행된다 — 컷씬 전환 연출에 데미지 숫자가 묻히지 않게 하기 위함.

## 5. 발동 중 이동 효과 — `ApplyBoost`

매 프레임 **서버**와 **소유 클라** 양쪽에서 재적용(다른 스킬 종료가 `speedMultiplier`를 되돌려도 취소되지 않게).

| 효과 | 내용 |
|---|---|
| 속도 부스트 | `PlayerController.speedMultiplier = DashSpeedMul(1.4)` **+** `WalkSpeed`/`FallSpeedMaxX` 상한도 ×1.4 (상한을 같이 안 올리면 엔진이 원래 속도로 clamp) |
| Y축 하강 잠금 | `Gravity = 0`, `FallSpeedMaxY = 0` → 일정 높이로 글라이드(궁극기 중 하강 없음). 잔여 점프 Y속도는 `SetForce`로 0 |
| 점프 시 중력 복귀 | 발동 중 점프키 → `ReleaseGravityLock` + 서버 미러(`RequestReleaseGravity`) → 이후 중력/점프 아크 정상 |

종료 시 `EndBoost`(idempotent)가 스냅샷값(`baseWalkSpeed`/`baseFallSpeedMaxX`/`baseGravity`/`baseFallSpeedMaxY`)으로 정확히 복원.

## 6. 전방 타겟 처리 — `DestroyTargetsInPath` (ServerOnly, 발동 중 매 프레임)

항상 **전방(+X)** 판정(러너는 우측 진행, 피격 넉백으로 LookDirection이 뒤집히는 문제 회피). 박스 중심 `wp.x + DashHitOffset(1.5)`, 크기 `DashHitWidth(4) × DashHitHeight(1.8)`.

| 패스 | 대상 | 감지 | 처리 |
|---|---|---|---|
| Pass 1 | 몬스터 | `_CollisionService:GetSimulator → OverlapAll(CollisionGroups.HitBox, BoxShape)` | `DestroyTarget` → `e:Destroy()` (즉시 삭제) |
| Pass 2 | 장애물(`script.Obstacle`, 콜라이더 없음) | 맵 자식 + 박스 거리 테스트 | `LaunchKnockback(dir)` (삭제 아닌 튕겨냄) |
| Pass 3 | 보스(`script.MushmomBoss`) | 타입+위치(HitBox 그룹 불일치 대비) | `ApplyExternalDamage`로 **틱뎀**(공격력 × `BossDamageMul(2)`, `BossDamageTickInterval(1s)`마다). **삭제하지 않음** — 정상 사망 연출 경로 |

- 몬스터 데미지 = `GetPlayerAttackPower()`(= `PowerStrike:GetEffectiveDamage()`). 실제로는 Pass1이 즉시 `Destroy`라 데미지 계산 없이 삭제.

## 7. 컷씬 종료 피니셔 (검기 10발)

컷씬 종료 후 `FinisherDelay`(0.35초) 뒤, `UltimateSkill.RequestCutsceneFinisher` → **`PowerStrike:FireFinisherBeams(FinisherHits, FinisherBeamInterval)`**.

- **평소 검기 파이프라인 그대로 재사용**: 각 검기는 `SwordBeamAttack.AttackFrom → HitEvent → MonsterUnit.HandleHitEvent`를 타므로 **감지·데미지·자동 데미지숫자·피격이펙트·처치크레딧·슬로우/도트/급소일격**이 일반 검기와 100% 동일.
- `FireFinisherBeams(count, interval)`: 첫 발 즉시 + `interval`(0.09s) 간격으로 `count`(10)발 전방 발사. 각 검기 관통 = **99**(앞 몬스터 전부 통과, 그룹 일괄 타격).
- **투명 검기**: 피니셔 검기는 `invisible=true`로 발사돼 **스프라이트만 숨긴다**(Launch 직후 `SpriteRendererComponent.Enable=false`). 히트 판정/데미지/데미지숫자/피격이펙트는 그대로 — 컷신이 시각 연출을 담당하고 검기는 판정만 한다. (스윙 사운드/몬스터 피격 스파크는 유지됨.)
- `FireBeam(direction, damage, lag, pierceOverride, invisible)` — 컷신 종료(`UltCutsceneHud.Finish`, 종료 시점 결정적) 지점에서 호출. ⚠️ mLua는 선언 파라미터 전부 넘겨야 하므로 모든 `FireBeam` 호출부가 5개 인자 전달.

> **알려진 밸런스 특성:** 검기 1발 데미지가 커서(예 360) 일반 몬스터(HP≈50)는 첫 발에 즉사한다. 따라서 단일 약몹엔 "10타"가 다 안 보이고, 10발 배럴은 **그룹/탱키 대상**에서 잘 드러난다. 각 몬스터가 10대를 다 눈에 맞게 하려면 검기당 데미지를 낮추는 별도 옵션이 필요(미구현).

---

## 8. 종료

- 서버 `OnUpdate`: `remaining -= delta`, 0 이하 → `EndDashServer`.
- `EndDashServer`(ServerOnly): `Active=false`, `EndBoost`(속도/중력 복원). `@Sync Active=false` 전파.
- `OnSyncProperty("Active", false)` → `EndDashClient`(이펙트 제거, 소유 클라 `EndBoost`).
- 사망 시(`PlayerHealth.Die`)에도 `EndDashServer` 호출해 부활 팝업 중 돌진이 계속되지 않게 함.

## 9. 다른 시스템과의 상호작용

| 시스템 | 상호작용 |
|---|---|
| **Slide** | `Active`면 슬라이드 억제(궁극기 중 슬라이드 X) |
| **자동공격(PowerStrike/SkillManager)** | `ShouldAutoActivate`가 `ult.Active`면 false → 검기 자동발사 억제(궁극기 중 자동공격 X) |
| **RunGameHud** | 거리 게이지 충전(`RequestDistanceGauge`), 스테이지 시작 시 `ResetUltimate`/`ResetGauge` |
| **MushmomBoss** | 등장 시 `SetAutoFillPaused(true)`, 피격 시 `AddGauge`, 돌진 틱뎀 `ApplyExternalDamage` 수신 |
| **PlayerHealth** | HP 0 사망 시 `EndDashServer`로 돌진 강제 종료 |

`ResetGauge`(Server, 스테이지 시작): 발동 중이면 안전 종료, `Gauge=0`, `remaining=0`, `AutoFillPaused=false`(다음 스테이지에서 게이지 다시 차게).

## 13. 변경 이력(요약)

- 컷씬 종료 피니셔 추가: 초기 러너 정지형 → "멈추지 말라" 반영해 즉시 재개형 → 데미지 가시성 위해 `FinisherDelay` 지연 도입.
- 피니셔 몬스터 처리: `GetChildComponentsByTypeName`(0마리 버그) → 충돌 오버랩 → **최종: 검기 발사(AttackFrom 파이프라인)** 로 확정(감지·데미지숫자가 평소 검기와 동일).
