# 레벨 디자인(코스) 사양서 — 맵 배치 오브젝트

> 대상: TobenRun 러너 게임의 **스테이지(코스) 저작**
> 핵심: 레벨은 코드가 아니라 **데이터셋(Course 시트)에 "명령 행"을 적어** 몬스터/장애물/아이템/보스를 배치한다. 서버의 `CourseGen`(RootDesk/MyDesk/RunGame/CourseGen.mlua)이 스테이지 진입 시 그 행을 읽어 실제로 스폰한다.

---

## 1. 동작 개요

1. 플레이어가 스테이지 `1-1`에 진입 → `RunGameHud`가 `_CourseGen:RequestBuildCourse("1-1", map)` 호출.
2. `CourseGen`이 데이터셋 **`Course1_1`** 을 읽는다(스테이지 `1-1` → 시트 이름 `Course` + `1_1`, 하이픈→언더스코어).
3. 시트의 각 행을 명령으로 해석해 바닥/장애물/몬스터/아이템/보스를 스폰.
4. `Set` 명령으로 스테이지 지오메트리·속도·배경을 설정.

**데이터셋 종류**

| 시트 이름 | 용도 |
|---|---|
| `Course<stage>` (예: `Course1_1`) | 스테이지별 배치 명령 목록 |
| `Pattern<name>` (예: `PatternZigzag`) | 재사용 번들(묶음). `Place`로 코스에 삽입 |
| `MonsterTable` | 몬스터 종류 정의(id, maxHp, contactDamage, mesoReward, spriteRUID, patrolSpeed, jump* …) |
| `ObstacleDef` | 장애물 종류 정의(id → model, damage) |

> ⚠️ 시트를 수정하면 Maker **`refresh`** 후, 스테이지에 (재)진입해야 반영. 디버그 맵(BossTestMap)은 매 진입 강제 재빌드.

---

## 2. 시트 컬럼 & 좌표계

각 행은 **4개 컬럼**: `cmd`, `p1`, `p2`, `p3`

- **`cmd`** — 명령어(Set / Spawn / Gap / Place / Boss / Goal / Cutscene)
- **`p1`** — 명령 인자 1 (보통 종류/키)
- **`p2`** — **항상 x 좌표**(월드유닛, 코스 진행 방향). 코스 시작이 `startX`(기본 0)
- **`p3`** — 선택 인자(대개 바닥 위 높이 y, 또는 수량 override)

**좌표 규칙**
- x = 코스 시작(`startX`)부터의 월드유닛. 1유닛 = 100px.
- 바닥 Y = `floorY`(기본 0). `p3`(y)를 주면 대부분 `worldY = floorY + p3`(바닥 위 높이). 안 주면 종류별 자동 접지.
- 코스 실질 범위 = `[startX-4, startX+length+4]`.

---

## 3. 명령(Command) 레퍼런스

| cmd | p1 | p2 | p3 | 설명 |
|---|---|---|---|---|
| **Set** | 키 | 값 | — | 스테이지 설정(§4) |
| **Spawn** | 종류 id | x | (선택) 높이/수량 | 오브젝트 1개 배치(§5) |
| **Gap** | 구덩이 길이 | x | — | 바닥에 낙사 구덩이(x부터 length만큼 바닥 제거) |
| **Place** | 패턴 이름 | baseX | (선택) baseY | `Pattern<이름>` 번들을 baseX에 통째로 삽입(§6) |
| **Boss** | 보스 id | x | — | 고정 x 보스 배치(§7) |
| **Goal** | — | x | — | (클라 RunGameHud가 처리, 스폰 없음) |
| **Cutscene** | id | x | — | 예약(미구현) |

---

## 4. `Set` 키 목록

`Set,<키>,<값>` 형식. 여러 시스템이 나눠 읽는다(디자이너는 그냥 Set 행을 적으면 됨).

| 키 | 값 | 효과 |
|---|---|---|
| `startX` | 숫자 | 코스 시작 x(기본 0) |
| `length` | 숫자 | 코스 길이(기본 120) |
| `floorY` | 숫자 | 바닥 높이(맵 네이티브 풋홀드와 일치, 보통 0) |
| `goal` | 숫자 | 골인 x |
| `goal` | `boss` | **보스 아레나 모드**(고정 골 없음, 바닥 무한 연장 + 끝에서 보스 등장) |
| `speed` | 숫자 | 러너 달리기 속도(u/s) |
| `tile` | RUID | 바닥 타일 스프라이트 |
| `bg` | Template RUID | 스테이지 배경(BackgroundComponent) |
| `bgm` | 사운드 RUID | 스테이지 배경음 |
| `size` | 숫자 | (패턴 시트 전용) 번들의 x 점유폭 — 겹침 경고용 |

---

## 5. `Spawn` 오브젝트 카탈로그

`Spawn,<종류>,<x>[,<p3>]`. `<종류>`(p1)의 해석 우선순위: 특수 키워드 → 몬스터(MonsterTable) → 장애물(ObstacleDef) → 원시 modelId.

### 5.1 기본 픽업/지형

| 종류(p1) | p3 | 효과 |
|---|---|---|
| `coin` | 높이(기본 0.5) | 코인. 접촉 시 점수/골드 |
| `mud` | — | 진흙 웅덩이. 밟으면 감속(점프로 회피). 영구(러너 도달까지) |
| `sporewall` | 포자 데미지 override | **투명 포자 탄막 발사기**. 화면 앞에서 상/중/하 레인으로 포자 발사(점프 타이밍으로 회피) |
| `platform` | **y(바닥 위 높이)** | **밟는 발판**(공중 풋홀드). 플레이어가 더블점프로 올라설 수 있음. 예: `Spawn,platform,40,5` = x40, 바닥 위 5 높이의 발판. 너비/스프라이트는 CourseGen의 `platformWidth`(기본 3)·`platformRuid`(기본 바닥타일) 프로퍼티. 원웨이(아래에서 점프해 통과 후 윗면 착지) |

### 5.2 몬스터 (MonsterTable id)

`Spawn,<몬스터id>,<x>[,<높이>]` — `MonsterTable`에 있는 id면 몬스터로 스폰(공유 `normalmonster` 모델 + `MonsterUnit`이 그 id의 스탯/스프라이트/패트롤 적용).

| 알려진 id | 비고 |
|---|---|
| `normal` | 기본(슬라임) |
| `spore` | 스포아(제자리) |
| `mushmom` | 주황버섯(보스로도 사용) |

> 전체 id·스탯(HP/몸박 데미지/메소/패트롤 속도/점프)은 **`MonsterTable` 시트**에서 정의·확인. 새 몬스터는 시트에 행 추가.

### 5.3 장애물 (ObstacleDef id)

`Spawn,<장애물id>,<x>[,<높이>]` — `ObstacleDef`에 있는 id면 전용 모델로 스폰(`Obstacle` 스크립트가 스프라이트에서 히트박스 자동 크기, 접촉 데미지).

- 높이(p3) 미지정: 스프라이트 반높이만큼 바닥 표면에 접지(파묻힘 방지).
- 천장형(`IsCeiling`) 장애물은 `CeilingY`에 매달림.
- 알려진 예: `spike`(가시), `spike_up`(천장 가시) 등 — **전체 id·모델·데미지는 `ObstacleDef` 시트** 참조.

### 5.4 로그라이트 아이템 (스테이지 한정 버프)

`Spawn,<아이템>,<x>[,<수량override>]` — 획득 시 스테이지 한정 버프(스테이지 이탈 시 소멸). p3로 지급량 조절.

| 종류(p1) | 효과 | 기본 지급량 |
|---|---|---|
| `atkbuff` | 공격력 증가 | 0.5 |
| `aspdbuff` | 공격속도 증가 | 0.3 |
| `piercebuff` | 검기 관통 수 +1 | 1 |
| `regenbuff` | 초당 HP 재생 | 5/s |
| `maxhpbuff` | 최대체력 증가 | 50 |
| `shieldbuff` | 보호막 충전(피격 1회 무효) | 1 |
| `healbuff` | 즉시 회복(사실상 풀피) | 9999 |
| `magnet` | 자석(코인 흡입) 지속 | 6초 |
| `slowsword` | 슬로우검(검기 명중 시 감속) | (PowerStrike.SlowStrength) |
| `dotsword` | 도트검(검기 명중 시 지속뎀) | (PowerStrike.DotPerStack) |
| `ultgauge` | 궁극기 게이지 충전(%) | 10 |
| `invincible` | 무적 지속 | (PlayerHealth.InvincibleDuration) |
| `hitbuff` | 타수(다단히트) 증가 | (PowerStrike.HitPerStack) |

> 각 아이템 스프라이트 RUID는 `CourseGen`의 인스펙터 프로퍼티(`atkBuffRuid` 등)에서 교체 가능.

### 5.5 원시 modelId

위 어디에도 안 걸리면 `p1`을 **등록된 modelId**로 보고 그대로 스폰(커스텀 엔티티). `worldY = floorY + (p3 or 1.0)`.

---

## 6. 패턴(Pattern) 번들

반복되는 배치 묶음은 `Pattern<이름>` 시트에 한 번 정의하고 `Place`로 여러 곳에 삽입한다.

- 코스 행: `Place,<이름>,<baseX>[,<baseY>]`
- `Pattern<이름>` 시트: 일반 Spawn/Gap 행을 **상대 x**(0부터)로 작성. 삽입 시 각 행의 x에 `baseX`가 더해진다. `baseY` 주면 번들 전체를 그만큼 들어올림.
- 패턴 시트에 `Set,size,<n>` 행으로 점유폭을 선언(겹침 경고용).
- **중첩 금지**: 패턴 안에서 또 `Place` 불가(1단계만).

---

## 7. 보스 배치

두 가지 방식:

1. **고정 x 보스** — `Boss,<보스id>,<x>` (골이 숫자인 일반 코스). 빌드 시 즉시 스폰하지 않고, 플레이어가 접근하면 경고음 → 하늘에서 낙하 스폰.
2. **보스 아레나** — `Set,goal,boss` + `Boss,<보스id>,<x>`. 고정 골 없이 바닥이 무한 연장되고, 러너가 끝(`startX+length`)에 도달하면 보스 인트로 컷신 후 등장.

> 보스 id는 현재 `mushmom`만 정의(→ `MushmomBoss` 모델). 스탯은 `MonsterTable`의 `mushmom` 행. 새 보스는 `BossModelForId`에 분기 추가 필요.

## 9. 주의사항 (silent-fail 방지)

- **p2는 항상 x**. `Gap`도 `p1=길이, p2=x`. 순서 헷갈리면 엉뚱한 곳에 생성.
- **Refresh 필수**: 시트 편집 후 Maker `refresh` + 스테이지 재진입. 로그 `[CourseGen] built stage=... cmds=N`로 확인.
- 몬스터 id는 `MonsterTable`, 장애물 id는 `ObstacleDef`에 있어야 인식. 없으면 원시 modelId로 오인 → `unknown type/model` 로그.
- `sporewall`(탄막 발사기)와 `spore`(스포아 몬스터)는 **다른 것**. `spore`는 MonsterTable id.
- 구덩이(`Gap`)는 시각 바닥과 충돌 풋홀드가 같은 세그먼트로 뚫려 "보이는 구멍 = 실제 낙사 구멍"이 일치.
- 코스 엔티티 이름 접두어(`Monster_`/`Obstacle_`/`Coin_`/`Spore_`/`Mud_`/`AtkBuff_`/`Boss_`/`Foothold_`/`FloorVis_`/`Pit_`/`Spawn_`)는 `CourseGen`이 재빌드 시 정리 대상 판별에 사용 — 임의 변경 금지.

## 10. 확장 방법 요약

| 하고 싶은 것 | 방법 |
|---|---|
| 새 스테이지 | `Course<stage>` 시트 생성 → 명령 행 작성 |
| 새 몬스터 종류 | `MonsterTable`에 행 추가 → 코스에서 `Spawn,<id>,x` |
| 새 장애물 종류 | 전용 `.model` 저작 → `ObstacleDef`에 `id,model,damage` 행 → `Spawn,<id>,x` |
| 재사용 배치 묶음 | `Pattern<이름>` 시트 작성 → `Place,<이름>,baseX` |
| 새 보스 | 보스 `.model` + `BossModelForId` 분기 + `MonsterTable` 스탯 행 |
